### 2. KEY PARAMETERS & DEFINES

**LV1 Cache** (`design/lv1/def_lv1.sv`):
- **256 KB, 4-way set-associative** (ASSOC_WID=2, ASSOC=4)
- Address: 32 bits → Tag[31:16] (16 bits), Index[15:2] (14 bits), Offset[1:0] (2 bits)
- Data width: 32 bits; Cache line: 32 bytes (`CACHE_DATA_WID = 1<<(2+3) = 32`)
- MESI state: 2 bits per line; cache line = tag(16) + mesi(2) = 18 bits tag+mesi
- LRU: 3-bit pseudo-LRU per set (`LRU_VAR_WID = (1<<2)-1 = 3`)
- `NUM_OF_SETS = 1<<14 = 16384`; `CACHE_DEPTH = 1<<(14+2) = 65536` blocks total
- ICache/DCache boundary: `IL_DL_ADDR_BOUND = 32'h3FFF_FFFF` (addr[31:30]==0 → ICache)

**LV2 Cache** (`design/lv2/def_lv2.sv`):
- **8 MB, 8-way set-associative** (ASSOC_WID=3, ASSOC=8)
- Address: 32 bits → Tag[31:20] (12 bits), Index[19:2] (18 bits), Offset[1:0] (2 bits)
- Same data width (32), MESI (2-bit), cache line 32 bytes

### 4. MEMORY MODEL (`gold/memory.sv`)

- **Sparse associative array**: `reg [31:0] mem[int]` (integer-indexed, only stores written locations)
- **Read**: If address exists in mem, return stored value. If not, return default pattern: `32'h5555_AAAA` when `addr[3]==1`, `32'hAAAA_5555` when `addr[3]==0`
- **Write**: `mem[addr] = data_bus_lv2_mem`, asserts `mem_wr_done` next cycle
- **Read response**: Drives `data_bus_lv2_mem_reg` and asserts `data_in_bus_lv2_mem` for 1 cycle (all @posedge clk, 1-cycle latency)

**Arbiter** (`gold/lrs_arbiter.sv`):
- LRS (Least Recently Served) round-robin arbiter for the shared bus
- Maintains a 4-entry queue tracking service order
- Priority: processor requests first (via LRS queue), then snoop requests (fixed priority 0>1>2>3), then LV2 request
- Grants are combinationally ANDed with requests; if grant drops (req goes away), all grants cleared

**Architecture**: The Python TB runs **entirely in pure Python** using the custom `uvm_shim.py` framework. The "DUT" is `MesiCacheModel` — a behavioral Python model. There is **no RTL co-simulation** (no cocotb, no Verilator, no VPI/DPI). The driver calls `model.read()/write()` directly instead of wiggling signals.

**What's MISSING or BROKEN:**

1. **No RTL co-simulation**: The Python TB only tests the behavioral model against itself. It does NOT drive the actual Verilog RTL. To be a real verification environment, it would need:
   - A cocotb-based top (or pyuvm + cocotb) that compiles the SV RTL with a simulator
   - A proper BFM/driver that wiggles `cpu_rd`, `cpu_wr`, `cpu_rden`, `cpu_wren`, `addr_bus`, `data_bus` signals following the protocol in `cpu_driver_c.sv` (assert rd → next cycle assert rden → wait for data_in_bus → deassert)
   - A proper monitor that samples RTL signals (data_in_bus_cpu_lv1, cpu_wr_done, data_bus)
   - A system bus monitor equivalent to `system_bus_monitor_c.sv`

2. **Scoreboard has dual reference models**: Both `cache_scoreboard.py` and `mesi_cache_model.py` contain independent MESI models. The scoreboard compares the model's output against *its own internal model* — essentially model-vs-model, not model-vs-RTL. This means bugs would only be caught if the two Python models diverge.

3. **LRU logic inconsistency in `mesi_cache_model.py`**: The ICache LRU update for way 3 uses `s.lru = (s.lru & 0b001) & 0b010` which always yields 0 — this doesn't match the SV scoreboard's `lru[2]=0; lru[0]=0` logic precisely (SV preserves bit 1, Python clears it).

4. **Missing test coverage**: Only 2 tests ported (base_test, five_trans_test). The SV testbench has 25 tests including LRU eviction, MESI full transitions, dirty eviction, snoop sequences, stress tests, APB tests, etc. None of these advanced tests are ported.

5. **No assertion checking**: The SV interfaces (`cpu_lv1_interface.sv`, `system_bus_interface.sv`) contain ~20 SVA assertions (simultaneous rd/wr, timeout, signal stability, one-hot grants, etc.). None are replicated in Python.

6. **No APB/MESI-LRU monitoring**: The SV TB has `cpu_apb_interface`, `lv2_apb_interface`, `cpu_mesi_lru_interface` for deep observability. The Python TB has none of this.

7. **`requirements.txt` mismatch**: Lists `pyuvm>=3.0` but the code uses the custom `uvm_shim.py` and doesn't import pyuvm anywhere.

8. **No virtual sequences**: The SV TB has `virtual_sequencer_c` and `virtual_seqs.sv` for coordinating multi-core sequences. The Python TB doesn't have an equivalent — `BaseTest` just does `asyncio.gather()` manually.
